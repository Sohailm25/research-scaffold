# AGENTS.md - Research Scaffold Harness

## Scope

This file governs all work within `research-scaffold/`. This is the harness repository, not an experiment instance. Experiment instances are generated by `scaffold init` and have their own AGENTS.md.

## Mission

Build and maintain the research-scaffold harness: a reusable, ML/AI-focused autonomous research system that lets AI agents run experiment phases, check gates, and advance through the research lifecycle with epistemic guardrails.

## Architecture

Three layers:

1. **Orchestrator** (coordination): Loads config, determines current phase, dispatches work, evaluates gates
2. **Agent Runner** (execution): Prepares workspace, renders WORKFLOW.md prompt, dispatches to backend
3. **Agent** (work): Operates within the experiment's AGENTS.md contract

## Development Standards

- TDD: write failing tests first, then implement
- All code in `scaffold/` package
- All templates in `scaffold/templates/`
- Match patterns from `resattn/AGENTS.md` for experiment templates
- Use `.venv` for all Python execution
- Run on MacBook M4 with MPS backend

## Key Invariants

- Gate evaluation happens in the orchestrator, never in the agent (prevents self-gaming)
- AGENTS.md.j2 preserves all 8 epistemic standards verbatim from resattn
- State persists in JSON files, not databases
- Every experiment is a standalone git repo with beads tracking
- Negative results are valid publishable outcomes

## Testing

- `pytest tests/` for all unit tests
- Tests must cover config parsing, gate evaluation, state transitions, template rendering, init output
- Never mock behavior that matters for correctness

## Issue Tracking

This project uses **bd (beads)** for issue tracking.
Run `bd prime` for workflow context.

**Quick reference:**
- `bd ready` - Find unblocked work
- `bd create "Title" --type task --priority 2` - Create issue
- `bd close <id>` - Complete work
- `bd sync` - Sync with git (run at session end)
